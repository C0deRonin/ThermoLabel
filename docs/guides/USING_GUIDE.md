# Руководство по использованию Infralabel

Подробное описание интерфейса разметки термальных изображений. Репозиторий может называться **ThermoLabel**, в окне приложения отображается **Infralabel** — это одно и то же.

**Связанные материалы:** [Быстрый старт (Docker)](./QUICKSTART.md) · [Примеры API](../api/API_EXAMPLES.md)

## 1. С чего начать

1. Запустите приложение так, как описано в **[QUICKSTART.md](./QUICKSTART.md)** (обычно `docker compose up` и открыть URL фронтенда в браузере).
2. Убедитесь, что **backend** и **PostgreSQL** доступны: список проектов и сохранение читают данные из **API** (база), а не только из браузера.
3. Откройте интерфейс: по умолчанию вы на вкладке **«Аннотация»**, в центре — изображение (в т.ч. демо-снимок, если свой ещё не загружен).

---

## 2. Карта экрана (куда смотреть)

| Зона | Что там |
|------|---------|
| **Верхняя строка (шапка)** | Название файла, размер кадра **W×H**, **имя проекта** (нажмите, чтобы переименовать), переключатель **вкладок**, **Отменить / Повторить**, **Сохранить проект**, кнопки экспорта, **Загрузить изображение**, **Проекты**, **Дамп БД**, переключатель **темы** и **языка** (RU/EN). |
| **Левая колонка** | Инструмент, палитра, выбор **класса**, блок **«Курсор»** (координаты и температура под указателем), **гистограмма**. |
| **Центр** | На вкладке «Аннотация» — холст с изображением и аннотациями; снизу по центру — счётчик объектов, **масштаб** и флажок **«Тепловая карта»**. |
| **Правая колонка** | Список всех аннотаций с температурами и размером области. |

**Вкладки в шапке (главная навигация по режимам):**

- **Аннотация** — разметка на изображении.
- **Аналитика** — сводка по текущим аннотациям и классам (без редактирования).
- **Классы** — создание и редактирование классов (имя, цвет, диапазон температур).

## 3. Вкладка «Аннотация»

### 3.1. Загрузка изображения

1. Нажмите **«+ Загрузить изображение»**.
2. Выберите режим:
   - **С палитрой (тепловое изображение)** — картинка переводится в оттенки серого и дальше раскрашивается выбранной **палитрой** (удобно для имитации тепловизора).
   - **Без палитры** — пиксели используются как есть (без принудительной палитры).
3. Укажите файл. После загрузки **аннотации сбрасываются** — это нормально для нового кадра.

**Замечание:** прямой ввод файлов специальных форматов FLIR (`.fff`, `.seq`, `.raw`) может потребовать обработки на backend; при ошибке приложение может подставить демо-данные — ориентируйтесь на сообщение в браузере.

### 3.2. Выбор класса перед разметкой

В левой панели в блоке **«Класс»** выберите тип объекта. Цвет и подпись на холсте соответствуют этому классу. Новые фигуры получают **тот класс, который выбран в момент рисования**.

### 3.3. Отмена действий

- **↩ Отменить** и **↪ Повторить** в шапке откатывают изменения списка аннотаций (цепочка истории).
- Удаление одной фигуры или **очистка всего списка** делается в **правой панели** (см. ниже).

## 4. Инструменты разметки

Инструменты переключаются в левой панели в блоке **«Инструмент»**.

### 4.1. Bounding Box (прямоугольник)

1. Выберите **▭ Bounding Box**.
2. Зажмите левую кнопку мыши и потяните прямоугольник.
3. Отпустите кнопку. Прямоугольник добавляется, если его размер **не меньше ~8×8 пикселей** (очень маленькие клики игнорируются).

**Выделение:** клик по уже нарисованной фигуре выделяет её (повторный клик по той же — снимает выделение). Подпись на фигуре: номер, класс, средняя температура в области.

### 4.2. Полигон

1. Выберите **⬡ Полигон**.
2. **Кликайте** по вершинам многоугольника по очереди.
3. **Замкнуть контур** можно так:
   - кликнуть **рядом с первой точкой** (в радиусе примерно 14 px от неё), **или**
   - сделать **двойной клик** по холсту при ≥ 3 точках.
4. Пока контур не замкнут, доступны ссылки **«Закрыть»** и **«Сброс»** под списком инструментов (закрыть вручную / стереть точки).

Минимум **3 точки** для валидного полигона.

### 4.3. Порог (авто-области по температуре)

1. Выберите **⚡ Порог**.
2. Ползунком задайте температуру **от −20 °C до 120 °C**. На изображении подсвечиваются пиксели **не ниже** этого порога (визуальный ориентир).
3. Нажмите **«Применить авто»**. Алгоритм находит связные области выше порога и добавляет для них **прямоугольники** (класс — текущий выбранный). За один проход добавляется **не больше 25** новых областей; совсем крошечные регионы отбрасываются.

**Важно:** в режиме порога **не рисуется** обычный bbox мышью — сначала настройте порог и нажмите «Применить авто».

## 5. Палитра, зум и отображение

### 5.1. Палитра

В блоке **«Палитра»** выберите схему раскраски по интенсивности (например **IRON**, **RAINBOW**, **HOTSPOT** и др.). Пункт **ORIGINAL** показывает исходные RGB-значения буфера без перекраски.

Палитра и набор классов могут **сохраняться локально** в браузере и синхронизироваться с backend при доступном API.

### 5.2. Масштаб

- Кнопки **+** и **−** под холстом меняют масштаб от **40%** до **250%**.
- **Ctrl + колёсико мыши** на области изображения — то же самое.

### 5.3. Тепловая карта

Флажок **«Тепловая карта»** под холстом включает дополнительный слой визуализации по уже нарисованным **bbox** (наглядное «разогревание» вокруг объектов). Это **не** отдельная аннотация, а режим просмотра.

### 5.4. Блок «Курсор»

При наведении на изображение показываются координаты **X, Y** и расчётная температура **T** в точке (по данным кадра).

### 5.5. Гистограмма

Внизу левой панели — распределение значений по текущему кадру (ориентир при подборе порога и палитры).

---

## 6. Правая панель — список объектов

- Список идёт **сверху вниз** от последних добавленных к ранним.
- **Клик по строке** — выделить объект на холсте (ещё раз — снять выделение).
- Кнопка **✕** у строки удаляет только эту аннотацию (с записью в историю отмены).
- **«Очистить всё»** внизу удаляет все фигуры сразу.

В каждой строке отображаются: тип (▭ или ⬡), класс, **средняя** и **макс.** температура в области, габариты в пикселях.

---

## 7. Вкладка «Аналитика»

Здесь показывается **статистика по текущему набору аннотаций** на открытом изображении:

- карточки: число аннотаций, число задействованных классов, общий min/max температур;
- **распределение по классам** (доли и полоски);
- **средние/макс. температуры по классам**;
- **scatter** — условное расположение центров областей на кадре.

Переключение на эту вкладку **не сохраняет** проект автоматически — при необходимости сохраните вручную.

---

## 8. Вкладка «Классы»

Здесь задаётся **справочник классов** для разметки.

- Для каждого класса показываются **имя**, **цветной квадрат**, **диапазон температур** (°…°) и число аннотаций с этим классом.
- **Добавить класс:** блок **«ДОБАВИТЬ КЛАСС»** — введите название, при необходимости выберите цвет, нажмите **«+ Добавить»**. Новому классу автоматически задаётся диапазон **−20…120 °C** (как в коде).
- **Удалить класс:** кнопка **✕** у строки (доступна, если классов больше одного). Удаляйте класс только если уверены, что он больше не нужен в разметке.
- Отдельного поля для **смены диапазона температур** у уже существующего класса в интерфейсе нет — диапазон важен для отчётов и подписей, исходные классы из набора по умолчанию уже имеют свои интервалы.

Классы сохраняются в **localStorage** и при работающем API — на **сервере**.

## 9. Проекты: сохранение и загрузка

### 9.1. Имя проекта

В шапке отображается текущее имя. Нажмите на него — появится поле ввода; **Enter** сохраняет текст, **Escape** отменяет редактирование.

### 9.2. Сохранить

**«💾 Сохранить проект»** отправляет на backend:

- имя проекта;
- массив **сырых пикселей** изображения, размеры;
- текущую **палитру**;
- все **аннотации** и **классы**;
- метки времени.

Если API недоступен, сохранение через браузер **не подменяет** полноценное сохранение в БД — ориентируйтесь на сообщения об ошибке.

### 9.3. Меню «Проекты»

**«📁 Проекты»** открывает список записей из **базы** (через API):

- открыть проект — подгружает кадр и разметку;
- **создать новый** — сбрасывает состояние и создаёт новый проект с демо-изображением (см. алерт в интерфейсе);
- удалить проект из списка — с подтверждением.

Если список пуст, в подсказке указано: данные приходят из БД — **сохраните проект** или **импортируйте дамп**.

## 10. Экспорт датасета (YOLO, COCO, Pascal VOC)

Кнопки в шапке: **↓ YOLO**, **↓ COCO**, **↓ Pascal VOC**.

**Условия:**

- Должно быть **загружено изображение** (есть пиксельные данные).
- Должна быть **хотя бы одна аннотация**.

Иначе появится предупреждение о невозможности выгрузки.

**Поведение с базой данных:**

- Файл **всегда скачивается** в браузер.
- Если проект **уже сохранён** и имеет id в БД, текст экспорта дополнительно **сохраняется на сервере** (в таблице экспортов). У кнопки формата может появиться отметка **✓**.
- Если проект **не сохранён**, скачивание всё равно работает, но запись в БД **не выполняется** — в сообщении будет напоминание **сохранить проект**, чтобы экспорт попал в БД.

Для уже сохранённых экспортов доступны кнопки **«Скачать из БД»** рядом с подсказкой по форматам.

## 11. Дамп базы данных

Кнопка **«📦 Дамп БД»** открывает окно **экспорта и импорта** SQL-дампа PostgreSQL.

- **Полный дамп** — схема + данные (типичное резервное копирование).
- **Только данные** — удобно для **слияния** работ нескольких людей: каждый экспортирует «только данные», затем импортируйте файлы **по очереди**.
- **Импорт** — выбор файла `.sql` или `.dump`; для полного дампа можно включить опцию **очистки БД перед импортом** (осторожно: удалит текущие данные).

Подробности сценариев см. в подсказке внутри модального окна.

---

## 12. Тема, язык, несохранённые данные

- **Тема:** переключатель **☀️ / 🌙** в шапке (светлая / тёмная).
- **Язык:** выпадающий список **RU / EN** — переводит отдельные подписи интерфейса (кнопки дампа, экспорта, проектов и т.д.).
- **Закрытие вкладки:** если есть несохранённые изменения (есть загруженный контент или аннотации), браузер может показать стандартное предупреждение при уходе со страницы. Надёжно **сохраните проект** перед закрытием.

## 13. Советы и ограничения

| Совет | Зачем |
|-------|--------|
| Сначала настройте **классы** и **палитру**, потом массово размечайте | Единообразные цвета и отчёты. |
| Используйте **Отменить** при ошибочных кликах | Быстрее, чем удалять по одной в списке. |
| Для ровных полигонов кликайте точнее; замыкайте у **первой** точки или **двойным кликом** | Меньше «рваных» контуров. |
| Перед длинной сессией проверьте **Docker** и **доступность API** | Список проектов и сохранение завязаны на сервер. |
| Делайте **дамп БД** перед рискованными экспериментами с импортом | Можно откатить состояние базы. |

**Ограничения, о которых полезно помнить:**

- Очень маленький **bbox** не создаётся; у **порога** есть лимит числа авто-объектов за раз.
- **Тепловая карта** внизу экрана ориентирована на **bbox**, не на полигоны.
- Экспорт в файл **не заменяет** сохранение проекта в БД — это разные действия.
