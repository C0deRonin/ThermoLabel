# Руководство по использованию ThermoLabel

## 🚀 Начало работы

После открытия приложения на http://localhost:3000 вы увидите интерфейс ThermoLabel с:

### Три главных области:
1. **Левая панель** (196px) - инструменты и настройки
2. **Центр** - главный canvas с тепловизионным изображением
3. **Правая панель** (210px) - список всех аннотаций

And a header with controls and file upload.

---

## 📐 Режимы аннотации

### 1️⃣ Bounding Box (▭)
Самый быстрый способ разметить объекты прямоугольниками.

**Как использовать:**
1. Выберите инструмент "▭ Bounding Box" в левой панели
2. Выберите класс объекта (например, "Перегрев")
3. Нажмите и тащите мышь по изображению
4. Отпустите мышь - аннотация создана
5. Аннотация автоматически получит температурную статистику

**Советы:**
- Минимальный размер: 8x8 пиксели
- Отпущенные точки сразу сохраняются
- Клик на существующую аннотацию выделит её

### 2️⃣ Полигон (⬡)
Для разметки сложных форм и произвольных контуров.

**Как использовать:**
1. Выберите инструмент "⬡ Полигон"
2. Нажимайте на изображение чтобы добавлять точки
3. На панели слева увидите счетчик точек
4. Когда минимум 3 точки:
   - **Двойной клик** на первую точку, или
   - Нажмите кнопку "Закрыть" в левой панели
5. Полигон замкнется автоматически

**Советы:**
- Можно добавить неограниченное количество точек
- Нажмите "Сброс" чтобы начать заново
- Точки отображаются белыми кружками (первая) и цветными (остальные)

### 3️⃣ Пороговый инструмент (⚡)
Автоматическое выделение всех пикселей выше/ниже определенной температуры.

**Как использовать:**
1. Выберите инструмент "⚡ Порог"
2. Передвигайте слайдер чтобы выбрать температуру (-20°C до 120°C)
3. На изображении выберите белыми пиксели выше порога
4. Нажмите "Применить авто"
5. Автоматически создастся до 25 bounding boxes для всех горячих регионов

**Советы:**
- Отлично для быстрой разметки большого количества аномалий
- Автоматически пропускает регионы меньше 5x5 пиксели
- Каждый регион получит свой bbox с правильной температурной статистикой

## 🎨 Палитры отображения

В левой панели выберите палитру для визуализации тепловизионных данных:

- **IRON** (по умолчанию) - классическая палитра тепловизоров
- **RAINBOW** - спектр (синий → красный)
- **GRAYSCALE** - черно-белая
- **HOTSPOT** - подчеркивает горячие области
- **ARCTIC** - инверсия для холодных объектов
- **VIRIDIS** - наука-friendly перцептивно однородная

Все палитры одинаково работают - выбирайте что вам удобнее!

## 📊 Информация о курсоре

Наведите мышь на изображение и в левой панели увидите:

```
X    0..640
Y    0..480  
T    -20.0°C..120.0°C [в зависимости от выбранной палитры]
```

Этот значения обновляются в реальном времени и помогают выбрать нужный диапазон для порога.

## 📁 Работа с классами

### Добавить новый класс:
1. Нажмите вкладку "Классы" в верхнем меню
2. В секции "ДОБАВИТЬ КЛАСС":
   - Введите название (например, "Ледяная наледь")
   - Выберите цвет нажав на цветной квадрат
   - Нажмите "+ Добавить"
3. Новый класс сразу появится в левой панели

### Удалить класс:
1. На вкладке "Классы" найдите класс в списке
2. Нажмите кнопку "✕" справа
3. Класс удалится (после этого всем аннотациям этого класса нужно переменить класс)

### Свойства класса:
- **Название**: любое значение
- **Цвет**: используется для отображения на холсте и экспорте
- **Температурный диапазон**: -20°C до 120°C (используется как подсказка)

## Сохранение и экспорт

### Сохранить проект (локально)
- Нажмите "💾 Сохранить" вHeader
- Проект сохранится в браузер localStorage
- При перезагрузке страницы загрузится автоматически

### Экспорт в форматы

#### YOLO (для YOLOv5/v8)
```
Нажмите "↓ YOLO" скачает файл annotations.txt

Формат:
<class_id> <x_center_norm> <y_center_norm> <width_norm> <height_norm>
```

#### COCO JSON (для detectron2, mmdetection)
```
Нажмите "↓ COCO" скачает annotations_coco.json

Содержит:
- metadata об изображении (разрешение, имя)
- все категории классов с диапазонами температур
- все аннотации с температурной статистикой
```

#### Pascal VOC (для Faster R-CNN, TensorFlow)
```
Нажмите "↓ Pascal VOC" скачает annotations.xml

XML формат с:
- bounding boxes в абсолютных координатах
- температурной статистикой для каждого объекта
```

## 📈 Аналитика датасета

Нажмите вкладку "Аналитика" в верхнем меню.

### Основные метрики:
- **Аннотаций** - общее количество созданных аннотаций
- **Классов** - количество используемых классов
- **Max/Min темп** - температурные границы

### Графики:

**РАСПРЕДЕЛЕНИЕ КЛАССОВ**:
- Показывает сколько аннотаций каждого класса
- Бар-чарт с процентами
- Помогает выявить дисбаланс в разметке

**ТЕМПЕРАТУРЫ ПО КЛАССАМ**:
- Для каждого класса с аннотациями показаны:
  - `avg` - средняя температура
  - `max` - максимальная
  - `n` - количество элементов этого класса

**SCATTER: ПОЗИЦИИ АННОТАЦИЙ**:
- Интерактивный график расположения всех бокса на изображении
- Размер кружка = размер bbox
- Цвет = класс
- Клик на кружок = выбрать аннотацию

## 🔄 История редактирования

### Undo (↩)
- Нажмите кнопку "↩" в Header
- Отменяет последние добавления аннотаций
- Доступно пока была хотя бы одна операция

### Очистить всё
- На правой панели внизу нажмите "✕ Очистить всё"
- Удалит все аннотации из текущего снимка
- Это действие тоже можно отменить через Undo (но только один раз)

## 🎬 Загрузка изображений

### Поддерживаемые форматы:
- **Изображения**: JPEG, PNG, BMP, TIFF
- **Тепловизионные**: FLIR .fff, .seq, raw (+backend требуется)

### Как загрузить:
1. Нажмите "+ Фото" в Header
2. Выберите файл с компьютера
3. Изображение загрузится и отброситься на холсте
4. Размер холста автоматически подстроится

### FLIR форматы:
- Если загружаете .fff, .seq или .raw файл:
  - Фронтенд попытается обработать как обычное изображение
  - Для полной поддержки FLIR требуется backend
  - Запустите FastAPI сервер (см. README)

## 💡 Советы и трюки

### Быстрая разметка
1. Используйте пороговый инструмент для первого прохода
2. Вручную добавляйте/удаляйте недостающие/лишние боксы
3. Экспортируйте в YOLO

### Контроль качества
1. Используйте аналитику чтобы проверить распределение
2. Ищите аномалии (очень горячие/холодные)
3. Проверьте что все классы представлены

### Работа с большим датасетом
1. Не загружайте сразу все изображения
2. Работайте порциями (20-50 фото)
3. Сохраняйте проект после каждой порции
4. Экспортируйте в промежуточный формат (COCO)

### Визуализация
- Если палитра "железо" слишком монотонна - попробуйте "радугу"
- Если нужно найти горячие точки - переключитесь на "hotspot"
- Если контраст плохо виден - используйте "viridis"

## 🐛 Распространенные проблемы

**Q: "Не видно аннотацию после рисования"**
- A: Проверьте что выбран правильный класс (левая панель)
- A: Аннотация может быть очень маленькой - увеличьте масштаб кнопками +/-

**Q: "Не получается закрыть полигон"**
- A: Нужно минимум 3 точки
- A: Двойной клик должен быть именно на первую точку (белый кружок)

**Q: "Холсты не обновляется"**
- A: Обновите страницу (F5)
- A: Очистите localStorage и загрузите заново

**Q: "FLIR файл не загружается"**
- A: Требуется backend (FastAPI) для полной поддержки
- A: Как fallback используется демо-данные
